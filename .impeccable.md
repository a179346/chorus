# Chorus Design System

## Design Context

### Users
All developers who use Claude Code, from junior to senior. They want better session management — running multiple Claude sessions, switching between them, and keeping track of what each is doing. Context: focused coding sessions where cognitive load should be minimized. The app should serve power users without intimidating newcomers.

### Brand Personality
**Calm, crafted, minimal.** Like Linear — clean lines, quiet confidence, feels premium without trying too hard. The interface should feel like a well-made tool: invisible when you're in flow, delightful when you pause to notice the details.

Three words: **Quiet. Precise. Crafted.**

### Emotional Goals
- **Flow & focus**: The UI should disappear — users are immersed in their work, not managing windows. Minimal friction between intent and action.
- **Delight & polish**: Small details surprise and reward attention. The kind of craft that makes someone want to show it to a colleague.

### Aesthetic Direction
- **Reference**: Linear — systematic, dark-first, purposeful color, clear hierarchy
- **Anti-references**: Electron apps that feel like web pages. Overly decorative dashboards. Neon-on-dark "hacker" aesthetics. Generic Material/Bootstrap styling.
- **Theme**: Dark mode only. Cool-toned with blue as the primary accent. Monospace typography throughout (developer tool identity).
- **Density**: Compact but breathing — information-dense without feeling cramped.

### Design Principles
1. **Quiet until needed**: Chrome and controls recede; content and status lead. No decorative elements that don't serve a purpose.
2. **Status at a glance**: Session state, context usage, and activity should be immediately scannable without reading labels.
3. **Keyboard-first, mouse-friendly**: Every action reachable by keyboard. Mouse interactions feel natural but never required.
4. **Consistent motion language**: Transitions are fast (120-200ms), functional, and uniform. No bouncing, no elastic easing. Deceleration only.
5. **Earned complexity**: Start simple. Reveal depth through interaction, not upfront density.
