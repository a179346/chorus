# Chorus

Multi-session Claude Code terminal manager built with Electron + React.

## Design Context

### Users
All developers who use Claude Code, from junior to senior. They want better session management — running multiple Claude sessions, switching between them, and keeping track of what each is doing. Context: focused coding sessions where cognitive load should be minimized.

### Brand Personality
**Calm, crafted, minimal.** Like Linear — clean lines, quiet confidence, feels premium. Three words: **Quiet. Precise. Crafted.**

### Emotional Goals
- **Flow & focus**: The UI disappears — users are immersed in work, not managing windows.
- **Delight & polish**: Small details surprise and reward attention.

### Design Principles
1. **Quiet until needed**: Chrome and controls recede; content and status lead.
2. **Status at a glance**: Session state and activity should be immediately scannable.
3. **Keyboard-first, mouse-friendly**: Every action reachable by keyboard.
4. **Consistent motion language**: Fast (120-200ms), functional transitions. No bouncing.
5. **Earned complexity**: Start simple. Reveal depth through interaction.
